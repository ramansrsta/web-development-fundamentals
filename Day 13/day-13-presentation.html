Animations

Animations are a way to add movement and style to your website. They can be used to make your website more interactive and engaging.

Use keyframes to control intermediate states. Keyframes are used to specify the values for the animating properties at various points during the animation.

How it works ?
  • @keyframes defines animation steps.
  • animation applies them to elements.

Syntax of Animations

The animation property is a shorthand property for the following animation properties: animation: name duration timing-function delay iteration-count direction fill-mode;

  • name: Name of the @keyframes.
  • duration: Time taken for one cycle.
  • timing-function: Controls pacing (ease, linear).
  • delay: Wait time before starting.
  • iteration-count: Repeats (e.g., infinite).
  • direction: Reverse or alternate playback.
  • fill-mode: Controls end-state appearance.

Transition Basics

Why use it: Adds polish to state changes (hover, focus, etc.).

How Does It Work: Transition monitors changes and smoothly adjusts properties.

Common Transition Functions

rotate()

Rotates an element around its origin (center by default).

Syntax: transform: rotate(angle);

  • angle can be in degrees (deg), radians (rad), or turns (turn).
  • Example values: rotate(45deg), rotate(1rad), rotate(0.25turn).

Use Cases: Spinning animations (e.g., loading icons) or tilted designs.

Rotate Me!

scale()

Scales an element along the X and/or Y axes.

Syntax: transform: scale(sx, sy);

  • sx: Scale factor along the X-axis.
  • sy: Scale factor along the Y-axis (optional; defaults to sx).
  • Use scaleX() or scaleY() for single-axis scaling.

Use case: Zoom-in effects, enlarging images, hover effects.

Scale Me!

translate()

Moves an element along the X and/or Y axes

Syntax: transform: translate(tx, ty);

  • tx: Translation distance along the X-axis.
  • ty: Translation distance along the Y-axis.
  • Use translateX() or translateY() for single-axis movement.

Use case: Sliding animations, repositioning elements

Move Me!

skew()

Skews an element along the X and/or Y axes.

Syntax: transform: skew(ax, ay);

  • ax: Skew angle along the X-axis.
  • ay: Skew angle along the Y-axis (optional; defaults to 0).
  • Use skewX() or skewY() for single-axis skewing.

Use case: Artistic designs, dynamic text effects.

Skew Me!

Transform origin

Sets the pivot point for transformations.

Syntax: transform-origin: x y;

  • x and y can be keywords (center, top, left, etc.) or percentages.

Origin Me!

3D Transforms

Adds depth by manipulating the Z-axis.

Syntax:

  • rotateX(angle) and rotateY(angle): Rotates around the X or Y axes.
  • perspective(value): Adds depth perspective.

Differences Between Animations and Transitions

Animations:

  • Use keyframes to control intermediate states.
  • complex sequences or repeated effects.
  • Can be paused, reversed, or played at different speeds.

Transitions:

  • Simple property changes.
  • Single property changes.
  • Can be used for hover effects.

Common Timing Functions

Linear

  • Represents constant speed throughout the animation
  • No acceleration or deceleration
  • Movement is uniform and mechanical
  • Best used for: Simple, predictable movements
  • Visual representation: A straight diagonal line

Ease

  • Most balanced and commonly used timing function
  • Starts slow, speeds up in the middle, and slows down at the end
  • Creates a natural, smooth motion
  • Mimics real-world physics where objects don't start or stop instantly
  • Best used for: General-purpose animations, making transitions feel organic

Ease-in

  • Starts slowly and gradually accelerates
  • Initial movement is very gentle
  • Picks up speed towards the end of the animation
  • Best used for: Animations where you want a subtle, gradual start
  • Think of: An object slowly gaining momentum

Ease-out

  • Starts quickly and then decelerates
  • Begins with rapid movement
  • Slows down smoothly towards the end
  • Best used for: Dropping or dismissing elements
  • Think of: An object quickly approaching its final position and then gently settling

Ease-in-out

  • Combines ease-in and ease-out
  • Starts slowly, accelerates in the middle, and then decelerates
  • Most complex and natural-feeling timing function
  • Best used for: Complex, multi-stage animations
  • Creates the most fluid and sophisticated motion

Cubic Bezier

  • Advanced, fully customizable timing function
  • Defined by four control points
  • Allows precise control over animation's acceleration and deceleration
  • Syntax: cubic-bezier(x1, y1, x2, y2)
  • Example: cubic-bezier(0.25, 0.1, 0.25, 1) is similar to the standard ease

Accessible Animations

Key Principles
  • Avoid triggering vestibular disorders:
    • Use @media (prefers-reduced-motion) to respect user preferences.
  • Provide context:
    • Ensure animations communicate purpose without relying solely on visuals.
Examples of Accessible Animations
  • Example 1: Reduced Motion for Scroll Effects 
    @media (prefers-reduced-motion: reduce) {
          .scroll-effect {
              animation: none;
          }
      }
    Why it works: Stops motion-heavy effects for users with motion sensitivities.
  • Example 2: Subtle Animations for Feedback 
    .button {
          transition: background-color 0.3s ease;
      }
      .button:hover {
          background-color: darkblue;
      }
    Why it works: Adds interactivity without overwhelming the user.

Animation Best Practices for Accessibility

Do:
  • Use animations for feedback and focus indicators.
  • Respect user preferences for reduced motion.
  • Ensure animations communicate purpose.
Don't:
  • Use excessive animations that distract.
  • Rely on animations as the sole communication mechanism.
  • Use flashing or fast animations (can trigger seizures).
