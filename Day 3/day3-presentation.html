What are HTML Forms?

HTML forms collect user input and interact with servers

<form action="/submit" method="post">
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" required>
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <button type="submit">Submit</button>
</form>
Best Practice: Always use labels for accessibility

HTML Input Types

<!-- Text Inputs -->
<input type="text">      <!-- Single-line text -->
<input type="password">  <!-- Masked input -->
<input type="email">     <!-- Email validation -->
<input type="number">    <!-- Numeric input -->

<!-- Selection Inputs -->
<input type="checkbox">  <!-- Binary choice -->
<input type="radio">     <!-- Mutually exclusive options -->
<input type="file">      <!-- File upload -->
Pro Tip: Choose input types wisely for better user experience

HTML5 Form Validation

<!-- Built-in Validation Attributes -->
<input type="text" required>           <!-- Must be filled -->
<input type="number" min="0" max="100"> <!-- Number range -->
<input type="text" maxlength="50">     <!-- Max characters -->

<!-- Custom Validation -->
<input type="text" 
       pattern="[A-Za-z]+" 
       title="Letters only">
Validation improves data quality and user guidance

HTML Tables: Structuring Data

<table border="1">
    <thead>
        <tr>
            <th>Name</th>
            <th>Age</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>John Doe</td>
            <td>30</td>
        </tr>
    </tbody>
</table>
Use semantic tags like <thead> and <tbody> for better structure

Advanced Table Techniques

<!-- Merging Cells -->
<table>
    <tr>
        <td colspan="2">Spans two columns</td>
    </tr>
    <tr>
        <td rowspan="2">Spans two rows</td>
        <td>Cell 1</td>
    </tr>
</table>
Use colspan and rowspan for complex layouts

Forms Within Tables

<table>
    <tr>
        <th>Field</th>
        <th>Input</th>
    </tr>
    <tr>
        <td>Username</td>
        <td>
            <input type="text" required>
        </td>
    </tr>
    <tr>
        <td>Email</td>
        <td>
            <input type="email" required>
        </td>
    </tr>
</table>
Combine forms and tables for structured data entry

Best Practices

  • Always use semantic HTML
  • Implement client-side and server-side validation
  • Make forms accessible with proper labels
  • Use appropriate input types
  • Provide clear error messages
  • Consider responsive design
Good practices lead to better user experience
